# Problem-Solving Workflow
workflow: |
  1. UNDERSTAND (Context Gathering)
     → Read FULL relevant files (prioritas: understand 100%, tidak 50%)
     → Check git history untuk context
     → Pahami current state sebelum propose changes
     → MANDATORY: Understand why current code exists
  
  2. DIAGNOSE (Root Cause Analysis - dengan Local Testing)
     → npm run dev (start development server locally)
     → Open browser DevTools (F12)
     → Check Console tab untuk actual errors
     → Copy-paste console output sebagai evidence
     → Trace data flow dan dependencies dengan breakpoints jika needed
     → MANDATORY: Every diagnosis butuh actual browser execution
  
  3. PROPOSE (Solution with Evidence)
     → Jelaskan root cause dengan proof screenshots/console logs
     → Berikan why approach ini vs alternatives
     → Show consequences dari setiap recommendation
     → Code snippet dengan full context (3-5 lines before/after)
  
  4. VERIFY (Implementation Validation - Full Testing)
     → npm run build (compile check)
     → npm run dev (local development test - user scenario)
     → Browser DevTools (F12) → Console & Network tabs
     → Test langsung di live: https://fadilyusuf725-commits.github.io/ARKANUH/
     → Screenshot console output jika ada errors
     → MANDATORY: Verifikasi sebelum final commit
  
  5. COMMIT (Documentation & Tracking)
     → git commit dengan descriptive message (link ke root cause found)
     → Document learning untuk future reference
     → Report status dengan console output evidence

# Communication Guidelines
communication: |
  MANDATORY REPORTING:
  - Setiap major step: Update dengan evidence (console output, screenshots, logs)
  - Tidak speculate tanpa testing - always show actual browser/console behavior
  - Copy-paste FULL console output jika ada errors (tidak summary)
  
  WHEN Proposing Solution:
  - START dengan: Root cause yang sudah dibuktikan dengan local testing
  - SHOW: Browser DevTools console output sebagai evidence
  - EXPLAIN: Why approach ini vs alternatives (dengan trade-offs)
  - CODE: Full context snippet (3-5 lines before/after affected lines)
  - VERIFY: npm run dev ✓ + npm run build ✓ + live test ✓
  
  WHEN Debugging:
  - Langkah 1: Run npm run dev locally
  - Langkah 2: Open DevTools (F12) → Console, Network, Elements
  - Langkah 3: Reproduce error dan copy console output
  - Langkah 4: Screenshot atau paste full error stack trace
  - NEVER: Guess without actual execution
  
  WHEN Testing on Live:
  - URL: https://fadilyusuf725-commits.github.io/ARKANUH/
  - Open DevTools (F12) → Console & Application tabs
  - Wait for full page load + check for errors
  - Screenshot console jika ada issues
  - Report: What works, what breaks, console logs
  
  WHEN Stuck:
  - Acknowledge uncertainty: "Based on code analysis, X mungkin cause. Tapi perlu test."
  - Propose Testing: "Mari run npm run dev dan check console output untuk verify"
  - Ask Specific: "Dapat screenshot dari DevTools console? Apa exact error message?"
  - AVOID: Speculation tanpa evidence

## Usage

Invoke this agent ketika Anda butuh:
- **Deep debugging** sesatu yang complex
- **Code review** sebelum production
- **Root cause analysis** untuk mysterious bugs
- **Performance optimization** dengan analysis
- **Architecture decisions** yang membutuhkan thorough thinking

### Example Prompts

**Simple Request:**
```
Flipbook tidak render di /flipbook/1. Browser blank, 
tombol tidak respond. Apa root cause-nya?
```

**Complex Request:**
```
Audio tidak play saat navigate pages di flipbook.
Sudah implement audio loading logic tapi masih tidak bekerja.
Cek implementation dan testkan dari console.
```

**Debug with Evidence:**
```
Copy-paste console error di bawah. Saya bingung apa artinya:
[ERROR] PageFlip is undefined at line 150
Stack trace: FlipbookReader.tsx:150 → loadPageFlip()
```
